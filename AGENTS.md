# AGENTS.md

Context for AI agents (and humans) working in this repo. RoExtract is a single-binary Rust desktop app (eframe/egui) that reads Roblox's on-disk cache and extracts assets by scanning file magic bytes. Cross-platform Windows/Linux; macOS is not supported.

## Build & run

- Dev build (fast compile, slow run): `cargo build`
- Release build (slow compile, fast run, <10MB binary via `lto = "thin"` + `strip = true` + `panic = "abort"`): `cargo build --release`
- Run GUI: `cargo run` (no args â†’ GUI, see `src/main.rs:147`)
- Run CLI: `cargo run -- --help` (CLI is work-in-progress; see `docs/en-GB/CLI.md` and the `Cli` struct in `src/main.rs:18`)

There is no `opencode.json` and no other instruction files in the repo; this is the only one.

## Tests

- Run all: `cargo test`
- Run a single test: `cargo test test_determine_category_png` (substrings match)
- Run one module's tests: `cargo test --lib logic::tests`
- Tests are unit tests inside `#[cfg(test)] mod tests` blocks in `src/logic.rs`, `src/locale.rs`, `src/logic/cache_directory.rs`, `src/logic/sql_database.rs`. Two of them create temp files under `std::env::temp_dir()` named `roextract_test*` and clean up after themselves; they assume the temp dir is writable.
- No integration tests, no test fixtures, no snapshot tests, no network tests. `cargo test` is safe to run offline.

## Lint / format / typecheck

- Standard Rust toolchain only. There is no `rustfmt.toml`, no `clippy.toml`, no configured lint overrides. Run `cargo fmt` and `cargo clippy` if you want checks; CI does not run them (`.github/workflows/build-and-release.yml` only does `cargo build --release`).
- The repo has two Python scripts (`build_readme.py`, `locale_sync.py`) with no virtualenv/requirements; they are standalone and only run manually.

## Generated code: do not edit by hand

- `build.rs` runs at compile time and generates `$OUT_DIR/locale_data.rs`, which `src/locale.rs` includes via `include!(concat!(env!("OUT_DIR"), "/locale_data.rs"))`. It bundles every `locales/*.ftl` file into a `get_locale_resources` function and a `LOCALES` const array. Any change to `locales/` is picked up automatically â€” `build.rs` emits `cargo:rerun-if-changed` per locale file.
- `build.rs` also embeds `COMPILE_DATE` as a rustc env var (used in the About tab) and, on Windows only, compiles `assets/icon.ico` into the binary's resources and sets the `asInvoker` manifest.
- Adding a new locale: drop a new `locales/<tag>.ftl` file (Fluent syntax). No other registration needed; `build.rs` picks it up. To sync missing keys into all other locales from `en-GB.ftl`, run `python3 locale_sync.py` (adds `# TODO: Translate` markers).

## README.md is generated

- Root `README.md` is generated by `python3 build_readme.py` from `docs/languages.md` + `docs/en-GB/README.md`. Its first line is a "do not edit" notice. Edit those two source files and re-run the script; do not hand-edit `README.md`.

## Architecture in one pass

- Entry: `src/main.rs`. parses CLI args via `clap`; with no flags it calls `gui::run_gui()`. On exit it always calls `config::save_config_file()` and `logic::clean_up()` (unless an update install script took over via `updater::run_install_script`, which `exit(0)`s the process).
- `src/logic.rs` is the core. `Category` enum (`Music`, `Sounds`, `Images`, `Ktx`, `Rbxm`, `All`) drives both GUI tabs and CLI `--mode`. Assets are identified by magic-byte headers in `get_headers()`; music is special â€” Roblox stores it raw with no HTTP header, so it has empty headers and the extension is detected from the file's own magic bytes (see `detect_file_extension`, defaulting to `ogg`).
- Three pluggable asset backends in `src/logic/`, each implementing `refresh`, `read_asset`, `create_asset_info`, `swap_assets`, `copy_assets`, `clear_cache`:
  - `cache_directory.rs` â€” Roblox's HTTP cache on disk. Default dirs: `%Temp%\Roblox` (Windows) and `~/.var/app/org.vinegarhq.Sober/cache/sober` (Linux/Vinegar Sober). Fs-reading path.
  - `sql_database.rs` â€” Roblox's SQLite asset DB (`rusqlite` with the `bundled` feature, so no system SQLite needed).
  - `rbx_storage_directory.rs` â€” Roblox's rbx-storage dir.
  - An `AssetInfo` has `from_file` / `from_sql` / `from_rbx_storage` flags; `read_asset` dispatches on whichever is true.
- `maybe_decompress` transparently zstd-decompresses cache files starting with the `0xFD2FB528` magic; callers don't need to know.
- `src/gui.rs` + `src/gui/{file_list,settings,welcome}.rs`: eframe/egui app, tabbed via `egui_dock`. Tabs are hard-coded strings (`"music"`, `"sounds"`, `"images"`, `"rbxm-files"`, `"ktx-files"`, `"settings"`, `"logs"`, `"about"`) passed to `locale::get_message` for display. `Alt`/`Ctrl`+`1..8` switches tabs.
- `src/updater.rs`: talks to `https://api.github.com/repos/AeEn123/RoExtract/releases/latest` (or `/releases` when prereleases enabled). **Always sets a `User-Agent: RoExtract (Rust)` header â€” GitHub returns 403 otherwise.** Background path (`check_for_updates_background`) is non-thread-blocking; the blocking path is for CLI only. Install scripts are `src/installer/installer.{sh,bat}`, included via `include_str!`, run via `sh`/`cmd /c` and then `exit(0)`.
- `src/locale.rs` + `locales/*.ftl`: Fluent-based i18n. `en-GB.ftl` is the reference; unknown locale keys fall back to returning the key itself, unknown locales fall back to `en-GB`.
- `src/config.rs`: two JSON configs. `RoExtract-config.json` (next to the binary, user state â€” gitignored) is mutable. `RoExtract-system.json` (next to the binary, shipped by package maintainers â€” see `packages/*/RoExtract-system.json`) is read-only and can override things like `config-path`, `allow-updates`, `prefer-installers`. Both are loaded into `LazyLock` globals at startup; `save_config_file()` flushes on exit.

## Conventions that differ from defaults

- Logging is not `log`/`tracing`. Custom macros in `src/log.rs` (`log_info!`, `log_warn!`, `log_error!`, `log_debug!`, `log_critical!`) append to a global `String` and also `println!`. `log_debug!` is a no-op in release builds. `log_critical!` additionally shows a native dialog (can block â€” do not call from the GUI render loop). `log_info!` etc. are `#[macro_export]`, so they're callable as `crate::log_info!` from anywhere.
- Many GUI-thread shared globals are `LazyLock<Mutex<...>>` in `src/logic.rs` (`STATUS`, `PROGRESS`, `FILE_LIST`, `FILTERED_FILE_LIST`, `REQUEST_REPAINT`, `LIST_TASK_RUNNING`, `STOP_LIST_RUNNING`, `TASK_RUNNING`). The asset lists are `Arc<Vec<AssetInfo>>` so the GUI can take a cheap copy-on-write snapshot per frame instead of cloning; mutate through `Arc::make_mut` (see `update_file_list`). The test `test_file_list_arc_snapshot_and_filter` asserts this contract â€” if you touch the list globals, keep that test passing.
- `windows_subsystem = "windows"` is set only when `debug_assertions` is false (`src/main.rs:1`). In debug builds on Windows you get a console window; in release you don't. Don't add `println!` for user-facing output in release; it goes nowhere on Windows.
- All locale strings go through `locale::get_message` â€” never hardcode UI text.
- Version is `CARGO_PKG_VERSION` (in `Cargo.toml`). Don't bump it in lockfile churn; the maintainer specifically calls out `cargo`/Gemini touching the version as a repeated annoyance (see recent commit messages).

## CI / release

- `.github/workflows/build-and-release.yml` triggers on pushes/PRs to `main` touching `**.rs`, `**.toml`, `**.ftl`, or `assets/**`. It builds Windows (`windows-latest`) and Linux (`ubuntu-latest`) release binaries and, on pushes to `main` (not PRs), publishes a prerelease tagged `dev-build-{run_id}` with `RoExtract-windows.exe` and `RoExtract-linux`.
- There is no stable-release workflow in-repo; stable releases are cut manually. The `delete-old-releases.yml` workflow trims old dev releases.
- Packaging: Arch PKGBUILD (`packages/arch/PKGBUILD`) and experimental Flatpak (`packages/flatpak/`, see its README). Both expect `cargo build --release` and ship a `RoExtract-system.json` next to the binary.
